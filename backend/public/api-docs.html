Laço — API Docs
Referência completa para integração com n8n, WhatsApp e automações
Autenticação — Todas as rotas /api/* exigem o header:

x-api-key: SUA_API_SECRET

Rotas marcadas com Público não precisam de autenticação.
Status dos Leads
captado
Preencheu nome/WhatsApp no portal por conta própria
link_sent
Clínica enviou o link de agendamento via WhatsApp
scheduled
Confirmou agendamento pelo portal
rejected
Recusou a oferta no portal
Status de Agendamentos
pending
Aguardando confirmação da clínica
confirmed
Confirmado pela clínica
done
Procedimento realizado
cancelled
Cancelado
⚡ Fluxo recomendado no n8n
1
Mensagem chega no WhatsApp → POST /api/leads com status: "link_sent" para registrar e já taguear
2
Envia o link de agendamento para o cliente via WhatsApp
3
Cliente abre o link → preenche dados → agenda (tudo automático pelo portal)
4
Polling ou webhook: GET /api/leads?status=scheduled para ver novos agendamentos
5
Confirmar presença: PATCH /api/appointments/:id/status com status: "confirmed"
6
Após procedimento: PATCH /api/appointments/:id/status com status: "done"

POST /api/leads Criar ou atualizar por telefone
Upsert por telefone — se já existir, atualiza nome e status. Use após enviar o link no WhatsApp.
CampoPadrãoDescrição
nameobrigatório—Nome do cliente
phoneobrigatório—Telefone só dígitos com DDD
sourceopcionalmanualwhatsapp · manual · link
statusopcionallink_sentStatus inicial do lead
Body
{
  "name": "Joana Silva",
  "phone": "11999999999",
  "source": "whatsapp",
  "status": "link_sent"
}
Resposta 201
{
  "id": "uuid",
  "name": "Joana Silva",
  "phone": "11999999999",
  "status": "link_sent",
  "created_at": "2025-01-15T14:30:00Z"
}
PATCH /api/leads/:id/status Atualizar status
CampoDescrição
statusobrigatóriocaptado · link_sent · scheduled · rejected
procedure_viewedopcionalUUID do procedimento visualizado
reject_reasonopcionalMotivo da recusa (texto livre)
Body
{
  "status": "link_sent"
}
DELETE /api/leads/:id Excluir lead e agendamentos
Remove também todos os agendamentos vinculados ao lead.
Resposta 200

POST /api/procedures Criar procedimento
CampoTipoDescrição
nameobrigatóriostringNome do procedimento
durationobrigatóriointegerDuração em minutos
priceobrigatóriointegerPreço em centavos
price_oldopcionalintegerPreço antigo em centavos
payment_noteopcionalstringNota de pagamento
video_urlopcionalstringLink do YouTube
Body
{
  "name": "Botox - Testa",
  "duration": 35,
  "price": 45000,
  "price_old": 60000
}
PATCH /api/procedures/:id Atualizar procedimento (parcial)
Envie apenas os campos que deseja alterar — os demais são preservados.
Body (exemplo — ativar/desativar)
{ "active": false }
Body (exemplo — atualizar preço)
{ "price": 50000, "price_old": 65000 }
